# 2. 行内语法

```markdown
正确：运行 `example --help` 查看可用参数
错误：运行`example --help`查看可用参数

正确：不要引入 `Spec`、`Options`、`Config`。
错误：不要引入 `Spec` 、 `Options` 、 `Config` 。

正确：配置项包括 `timeout`、`retry`；默认值使用 `false`。
错误：配置项包括`timeout`、`retry`；默认值使用`false`。

正确：请 **先确认范围** 再继续修改
错误：请**先确认范围**再继续修改

正确：参考 [使用说明](guide.md) 完成配置
错误：参考[使用说明](guide.md)完成配置
```

# 3. 标题层级

正确示例：

```markdown
# 1. 背景
## 1.1 当前问题

# 2. 方案
## 2.1 实施步骤
```

错误示例：

```markdown
# 1. 项目说明
## 1.1 背景
## 1.2 当前问题
## 1.3 方案
## 1.4 实施步骤
```

更细层级优先使用引用块：

```markdown
## 2.1 实施步骤

> 准备配置文件

准备配置文件的内容...

> 运行校验命令

运行校验命令的内容...
```

不要新增 3 级或更深标题：

```markdown
## 2.1 实施步骤
### 2.1.1 准备配置文件

准备配置文件的内容...
```
