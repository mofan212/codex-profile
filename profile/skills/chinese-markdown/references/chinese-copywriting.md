# 1. 中文文案排版规则

用于处理中文文案、产品文案、说明文档和 Markdown 正文中的非 Markdown 专属排版规则。只在任务涉及中文文案排版、标点、全角半角、数字单位、专有名词或缩写时读取本文件。

参考来源：[chinese-copywriting-guidelines](https://github.com/sparanoid/chinese-copywriting-guidelines/)

# 2. 空格

| 规则 | 正确 | 错误 | 说明 |
| --- | --- | --- | --- |
| 中文与英文之间加一个半角空格 | 在 LeanCloud 上运行 | 在LeanCloud上运行 | 产品名按官方写法保留，例如 `豆瓣FM` |
| 中文与数字之间加一个半角空格 | 花费 5000 元 | 花费5000元 | 数字前后按中文语境留白 |
| 数字与单位之间加一个半角空格 | 带宽为 10 Gbps，容量为 20 TB | 带宽为 10Gbps，容量为 20TB | 常规计量单位前留空格 |
| 度数和百分比不加空格 | 角度为 90°，提升 15% | 角度为 90 °，提升 15 % | `°` 和 `%` 直接跟随数字 |
| 全角标点前后不加空格 | 买了一部 iPhone，很开心！ | 买了一部 iPhone ，很开心！ | 中文全角标点与相邻字符紧邻 |

# 3. 标点和全角半角

| 规则 | 正确 | 错误 | 说明 |
| --- | --- | --- | --- |
| 中文语境使用全角中文标点 | 你好！今天可以提交吗？ | 你好! 今天可以提交吗? | 英文整句内部除外 |
| 中文引号使用直角引号 | 「请先确认范围」 | “请先确认范围” | 内层引号使用 `『』` |
| 不重复使用标点 | 这个结果需要复核！ | 这个结果需要复核！！ | 避免堆叠问号、叹号或混合标点 |
| 数字使用半角字符 | 价格为 1000 元 | 价格为 １０００ 元 | 设计稿中为对齐而使用全角数字时另行判断 |
| 完整英文句子内部使用半角标点 | 「Stay hungry, stay foolish.」 | 「Stay hungry，stay foolish。」 | 英文书名、报刊名和特殊英文名词同理 |

# 4. 名词

| 规则 | 正确 | 错误 | 说明 |
| --- | --- | --- | --- |
| 专有名词使用官方大小写 | 使用 GitHub 登录 | 使用 github 登录 | 不要自行全大写、全小写或混合大小写 |
| 技术名词按通行写法书写 | TypeScript、HTML5、React、Next.js | Ts、h5、RJS、nextjs | 避免不地道或团队外难以识别的缩写 |
| 英文书名和报刊名使用英文斜体 | 推荐阅读 *Hackers & Painters* | 推荐阅读《Hackers & Painters》 | 中文书名号不套英文标题 |

# 5. 检查清单

- 中文与英文、数字、常规单位之间已加一个半角空格
- 度数和百分比与数字紧邻
- 全角中文标点前后没有多余空格
- 中文语境使用全角标点和直角引号
- 没有重复堆叠标点
- 数字使用半角字符
- 完整英文句子、英文标题和特殊英文名词内部使用半角标点
- 专有名词大小写符合官方或通行写法
