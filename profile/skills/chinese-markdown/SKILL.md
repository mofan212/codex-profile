# 1. 中文 Markdown 编写规则

使用本 Skill 新增、编写、修改、格式化或审查中文内容的 Markdown / `.md` 文档：

1. 先遵守目标 Markdown 文档已有的结构、标题、列表、表格和行内语法约定；
2. 默认 **同时检查** Markdown 语法习惯和中文内容排版：先按本文件检查 Markdown 语法习惯，再读取 [chinese-copywriting.md](references/chinese-copywriting.md) 对中文内容排版进行检查；
3. 如果用户明确限定只检查某一类问题，例如只检查标题编号、链接格式或表格结构，则只处理用户指定范围；
4. 如果目标文档已有明确规则与本 Skill 不同，优先保持目标文档一致性。

# 2. Markdown 行内语法

- Markdown 行内代码、链接、强调等语法与中文内容相邻时，两侧留一个半角空格
- Markdown 行内语法与中文标点相邻时，不强制在标点前后加空格；行内代码后接顿号、逗号、句号、分号、冒号、问号或感叹号时，通常保持紧邻

```markdown
正确：运行 `example --help` 查看可用参数
错误：运行`example --help`查看可用参数

正确：不要引入 `Spec`、`Options`、`Config`。
错误：不要引入 `Spec` 、 `Options` 、 `Config` 。

正确：配置项包括 `timeout`、`retry`；默认值使用 `false`。
错误：配置项包括`timeout`、`retry`；默认值使用`false`。

正确：请 **先确认范围** 再继续修改
错误：请**先确认范围**再继续修改

正确：参考 [使用说明](guide.md) 完成配置
错误：参考[使用说明](guide.md)完成配置
```

# 3. 标题规则

文章内容最多只使用 1 级标题和 2 级标题。2 级标题必须属于前一个 1 级标题。

每个标题前都要指定序号，并按层级连续递增，例如：

```markdown
# 1. 标题
## 1.1 标题

# 2. 标题
```

不要只保留一个 1 级标题并把所有内容塞进 2 级标题。

正确示例：

```markdown
# 1. 背景
## 1.1 当前问题

# 2. 方案
## 2.1 实施步骤
```

错误示例：

```markdown
# 1. 项目说明
## 1.1 背景
## 1.2 当前问题
## 1.3 方案
## 1.4 实施步骤
```

需要更细层级时，优先使用引用块或列表，不要继续新增 3 级或更深标题。

正确示例：

```markdown
## 2.1 实施步骤

> 准备配置文件

准备配置文件的内容...

> 运行校验命令

运行校验命令的内容...

> 根据结果进行调整

根据结果进行调整的内容...
```

错误示例：

```markdown
## 2.1 实施步骤
### 2.1.1 准备配置文件

准备配置文件的内容...

### 2.1.2 运行校验命令

运行校验命令的内容...

### 2.1.3 根据结果进行调整

根据结果进行调整的内容...
```

# 4. 尾部标点

- Markdown 无序列表的条目末尾不要添加句号、分号、冒号、逗号、顿号等收尾符号
- Markdown 有序列表的条目末尾可以不加收尾符号；如使用收尾符号，非最后一项仅可用分号，最后一项仅可用句号
- 列表项作为小标题，并引出紧随其后的表格、代码块、引用块或子列表时，可以保留冒号；这类冒号属于结构提示，不算普通尾部标点
- Markdown 表格内文字末尾也不要添加这些收尾符号
- 只有条目或单元格内容本身必须保留原文标点时，才保留原文

正确示例：

```markdown
- 标题编号连续且层级正确
- 表格内文字末尾没有多余标点
- 命令和编码规则：

2. 修正标题编号；

```

错误示例：

```markdown
- 标题编号连续且层级正确。
- 表格内文字末尾没有多余标点；

1. 确认目标文档范围。
```

# 5. 输出和最终检查

交付前检查：

- Markdown 行内语法两侧与中文内容留有一个半角空格
- Markdown 行内语法与中文标点紧邻时没有误判为缺少空格
- 标题只使用 1 级和 2 级
- 标题编号连续且层级正确
- 无序列表条目末尾没有多余标点；有序列表未使用收尾符号，或仅在非最后一项使用分号、最后一项使用句号
- 引出后续表格、代码块、引用块或子列表的列表项可以保留冒号
- 表格内文字末尾没有多余标点
