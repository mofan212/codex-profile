# 2. 工作流程

| step_id | enter_condition | required_context | action | stop_when | verify |
| --- | --- | --- | --- | --- | --- |
| `collect_facts` | 需要实现、修改、重构或测试 Java 后端代码 | 相关源码、测试、配置、接口定义、调用方、同包类、父类或接口 | 使用 `rg` 定位类、方法、Bean、Mapper、Dto/DTO、Dao/DAO、枚举、异常和测试覆盖；先建立事实再设计方案 | 🔴 CHECKPOINT：需求、输入输出、兼容性或业务边界不明确且误判风险高 | 能说明读取了哪些文件，以及它们如何支撑方案 |
| `choose_local_style` | 已收集局部事实 | 当前代码库已有架构、命名、分层、异常处理和测试风格 | 优先贴合项目既有模式；缺少明确模式时再使用本 Skill 默认规范 | 事实与用户表述冲突 | 直接指出冲突并给出文件或代码依据 |
| `coordinate_naming` | 任务重点是设计、评审或重命名 Java 包、类、方法、变量或常量 | 当前命名对象及局部代码上下文 | 配合使用 `$java-naming`，不要自行另起命名体系 | 命名职责还未确定 | 先确定职责边界再选择名称或后缀 |
| `implement_minimally` | 事实和方案已明确 | 现有工具类、领域模型、接口契约、配置项和通用写法 | 只实现当前目标所需最小改动，保持简单直接，用早返回减少嵌套 | 方案开始引入非必要封装、抽象或模式化设计 | 每处改动都能追溯到用户目标或必要配套改动 |
| `protect_user_changes` | 工作区存在无关变更 | 当前任务相关文件和 diff | 不回滚用户未要求回滚的改动；只处理当前任务相关文件 | 🛑 STOP：无关变更阻塞当前任务且无法绕开 | 说明阻塞文件和需要用户决策的点 |

# 3. 失败处理与检查点

| trigger | first_action | fallback | stop |
| --- | --- | --- | --- |
| 需求、输入输出、兼容性或业务边界不明确，且误判可能改变接口行为、数据语义或持久化结果 | 🔴 CHECKPOINT：先向用户确认边界、示例数据或兼容要求 | 如果用户无法给出完整信息，只实现不改变接口行为、数据语义或持久化结果的安全子集 | 未确认前不实现高风险部分 |
| 当前项目约定与本 Skill 的参考规范冲突 | 采用当前项目约定，并在反馈中说明依据文件或代码位置 | 如果项目内存在多个冲突模式，优先跟随当前文件、同包类和直接调用链 | 不按参考规范强行改写 |
| 找不到相关测试、编译入口或必要运行条件缺失 | 先完成静态核对，说明未运行原因和已检查范围 | 如果能定位模块构建入口，只运行最小相关编译或测试；仍不可运行时列出已人工核对的接口、调用方和映射关系 | 不编造已验证结果 |
| 工作区存在无关变更且影响当前任务文件 | 基于当前 diff 协作，保留用户未要求回滚的改动 | 🛑 STOP：无法隔离改动时，说明阻塞文件、冲突片段和需要用户决策的点 | 未获得决策前不覆盖或回滚无关变更 |

# 4. 执行边界

- 先遵循当前仓库、当前模块和同包代码的既有模式；局部事实不足时，再使用本 Skill 的参考规范
- 只做用户目标所需的 Java 后端代码、测试或配置改动；不要顺手重构无关代码、扩大接口行为面或引入新架构风格
- 涉及 Java 包、类、方法、字段、变量、常量或后缀体系的设计和评审时，配合使用 `$java-naming`
- 参考规范与当前项目明确约定冲突时，优先当前项目约定，并在反馈中说明依据

# 5. 反例黑名单

| anti_pattern | forbidden | replacement |
| --- | --- | --- |
| 无事实编码 | 不要没读调用方、Mapper、DTO/DAO、测试和同包样本就直接实现 | 先用 `rg` 定位相关链路，并让方案能回到代码事实 |
| 顺手重构 | 不要重构无关类、调整无关格式、扩大接口行为面或引入新架构风格 | 只改用户目标和必要配套代码、测试、配置 |
| 过度防御 | 不要在上游已保证非空、非空集合或状态合法时重复堆叠兜底判断 | 只在外部输入、历史脏数据、第三方返回、反序列化或可空契约边界兜底 |
| 命名另起体系 | 不要自行引入新的 VO/BO/POJO/Support 后缀习惯 | 跟随当前项目，复杂命名配合 `$java-naming` |
| 验证空报 | 不要声称已验证但没有运行命令、测试结果或静态核对范围 | 明确命令和结果；无法运行时说明原因和已核对范围 |

# 6. 验证与反馈

| change_type | verify |
| --- | --- |
| 窄范围实现或修复 | 优先运行与修改范围最相关的单元测试、模块测试或编译命令，只扩大到必要模块 |
| Mapper / XML / DTO / DAO 改动 | 同步检查 Mapper 接口、XML、参数名、返回映射、调用方和相关测试 |
| 行为变化 | 补充聚焦单测，沿用当前模块已有测试框架和断言风格，覆盖新增分支、异常路径和兼容行为 |
| 只改注释、命名或局部格式 | 做静态核对，确认没有无关格式变更 |
| 测试或编译已运行 | 说明命令和结果 |
| 无法运行测试或编译 | 说明原因和已经完成的静态检查或人工核对范围 |

回复用户时使用中文，简明说明修改了什么、验证了什么、是否存在未完成风险。
