# 1. 兜底输入流程

本文件只在无法直接进入需求文档、AI 上下文入口或 AI 检索文档分支时读取。

# 2. 没有指定文档

如果用户没有指定任何文档：

1. 识别当前所在项目
2. 查看项目根、文档根或常见文档目录的文件列表，优先识别 `00-AI上下文入口.md`、`AI上下文入口.md`、`AI检索入口.md`、`*AI检索说明*.md`、`*AI检索文档*.md`、`*AI排查文档*.md`
3. 询问用户是否维护该项目下所有 AI 上下文入口和 AI 检索文档
4. 询问时说明将会检查哪些目录、是否会读取所有 AI 检索文档、是否会根据当前 git 变更补充维护

不要默认全量扫描和重写。

如果当前项目没有明显文档目录，先根据 `git status`、构建文件和目录名判断候选范围；仍无法判断时询问用户提供文档根目录。

# 3. 其他文档或错误触发

当用户指定了文档，但诉求明确是普通 Markdown 润色、表格排版、格式化或错别字修正时：

1. 说明该请求不属于 AI 检索文档维护
2. 不读取 AI 检索 reference，不生成 AI 检索说明
3. 退出本 Skill，按普通 Markdown 文档任务继续处理
4. 只有用户要求转为 AI 检索文档维护时，才重新进入本 Skill

当用户指定了文档，但它不是需求文档、AI 上下文入口或 AI 检索文档时，先询问用户，不要直接生成文档。

说明正确触发方式：

| input | next_action |
| --- | --- |
| 指定一个需求文档 | 结合代码变更生成或更新同目录 AI 检索说明 |
| 指定一个 AI 上下文入口 | 根据变更需求文档或路由范围统一维护 |
| 指定一个 AI 检索文档 | 向上找到入口并按入口维护 |
| 不指定文档但指定模块、功能、能力域、接口、类名、包名或业务流程，并要求维护 AI 检索类文档 | 先定位候选 AI 检索文档和建议路径 |
| 不指定文档 | 询问是否维护当前项目下所有 AI 上下文入口和 AI 检索文档 |
