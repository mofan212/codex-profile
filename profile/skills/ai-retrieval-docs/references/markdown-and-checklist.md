# 1. Markdown 规范

生成仓库内 Markdown 文档时遵守：

- 使用 `「」`，不要使用中文双引号。
- 中文与 English 之间加空格。
- Markdown 行内语法与中文内容之间加空格。
- 最多使用 1 级标题和 2 级标题。
- 标题需要编号并递增，例如 `# 1. 文档定位`、`## 1.1 背景文档`。
- 不要只写一个 1 级标题，然后把其他内容全部放到 2 级标题下。

# 2. 完成前检查

完成前检查：

- 是否说明读取了哪些文档和代码，以及为什么读取。
- 是否基于当前代码、测试和 diff，而不是只基于需求设想。
- 是否覆盖重要变更文件、入口链路、核心类、常量、配置、错误码和测试。
- 是否明确兼容性和边界。
- 是否维护了需求同级 AI 检索文档。
- 当同目录存在多个需求文档时，是否维护了需求目录入口。
- 当上层存在或需要顶层入口时，是否维护了上层 AI 上下文入口。
- 上下文入口是否适配所在项目 Agent 工作流和文档治理方式的适用部分。
- 文档改名或新增后，相关相对链接是否同步。
- 标题层级、编号、中文标点和中英文空格是否符合规范。
