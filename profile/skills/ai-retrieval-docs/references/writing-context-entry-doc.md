# 1. AI 上下文入口写作要求

本文件只在准备生成或更新 AI 上下文入口文档时读取。

AI 上下文入口应保持短小，主要负责路由、跳过依据和少量跨文档关键边界，不替代 AI 检索文档。

# 2. 推荐结构

根据项目实际裁剪；下列是候选结构，不是必填清单。加载路由是核心内容；其他章节只有能降低误读、重复加载或维护成本时才保留。

- `# 1. 文档定位`：说明本文是什么 context、能力域或目录的 AI 上下文入口，以及读取前置文档
- `# 2. 加载规则`：说明先筛选目录、能力域或需求主题，再按需加载
- `# 3. 工作流文档与长期文档边界`：如果项目存在 Agent skills、issue、PRD、临时工作区或长期文档目录，说明它们之间的边界
- `# 4. AI 检索说明更新规则`：说明哪些稳定事实变化必须更新，哪些格式或临时讨论不强制更新
- `# 5. ADR 规则`：如果项目使用 ADR，说明只为真实架构决策创建 ADR；文档组织规则通常不单独创建 ADR
- `# 6. 加载路由`：按任务目标、能力域或需求主题列出应读取的入口、AI 检索说明和必要补充文档
- `# 7. 核心事实`：只在存在明确误读风险时列出新旧关系、当前优先级和关键边界
- `# 8. 回答要求`：只在项目存在固定回答约束时，要求后续 AI 说明读取依据和跳过依据

项目没有对应工作流时，省略不适用章节，只保留加载路由和必要跳过依据。核心事实、回答要求、ADR 和工作流边界只有在存在明确误读风险、项目治理事实或固定回答约束时才保留。

写入或更新完成后，读取 [completion-checklist.md](completion-checklist.md) 进行检查。

# 3. 写作原则

- 按任务目标或能力域提供加载路由，不要机械罗列所有文件
- 清楚区分需求文档、过程文档、长期 AI 检索文档和上下文入口
- 说明后续 AI 应先读什么、何时补充读什么、哪些文档暂时无关
- 不复述详细实现，入口只保留路由、跳过依据和误读边界；细节回到 AI 检索文档或源码确认
- 路由要能解释跳过依据，例如某类任务只需读入口和一个 AI 检索文档，不需要读取同目录全部需求文档
- 更新时优先删除失效路由、合并重复提示和替换过期边界，不因新增文档而扩写实现背景
- 新增或改名文档后，同步维护相对链接和加载路由

# 4. 适配项目工作流

维护上层入口时，只沉淀所在项目已有且会影响加载决策的 Agent 工作流和文档治理方式，例如 context 入口、能力域或需求目录筛选、长期文档边界、AI 检索说明更新规则、ADR 规则、加载路由、核心事实和回答要求。

不要为了补齐结构写入不存在的项目治理规则、通用回答要求或实现背景。

若项目没有对应工作流、临时工作区或 ADR 体系，只保留适用部分并说明本项目现状。
