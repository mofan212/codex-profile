# 1. AI 上下文入口写作要求

本文件只在准备生成或更新 AI 上下文入口文档时读取。

AI 上下文入口应保持短小，只负责本层加载路由、跳过依据和少量跨文档关键边界，不替代上层承载文档或 AI 检索文档。

写入入口前，只在目标目录、入口路由和必要上层目录内识别已存在的上层承载文档。入口只引用这些文档的读取时机，不复述其职责、维护规则、回答要求或术语定义；若项目没有对应文档，不假设、不创建、不补齐该体系。

# 2. 推荐结构

根据项目实际裁剪；下列是候选结构，不是必填清单。加载路由是核心内容；其他章节只有能降低误读、重复加载或维护成本时才保留。

| 章节 | 用途 |
| --- | --- |
| `# 1. 文档定位` | 说明本文是什么 context、能力域或目录的 AI 上下文入口，以及前置文档读取时机 |
| `# 2. 加载规则` | 说明先筛选目录、能力域或需求主题，再按需加载 |
| `# 3. 工作流文档与长期文档边界` | 说明当前入口层特有的文档分流规则 |
| `# 4. AI 检索说明更新规则` | 说明当前目录特有的更新门槛 |
| `# 5. ADR 规则` | 说明当前能力域特有的 ADR 位置或例外规则 |
| `# 6. 加载路由` | 按任务目标、能力域或需求主题列出应读取的入口、AI 检索说明和必要补充文档 |
| `# 7. 核心事实` | 只在存在明确误读风险时列出新旧关系、当前优先级和关键边界 |
| `# 8. 回答要求` | 说明当前入口层特有的额外回答约束 |

「工作流文档与长期文档边界」、「AI 检索说明更新规则」、「ADR 规则」和「回答要求」只有在当前入口层存在额外规则，且未被上层承载文档说明时才保留；通用治理、回答要求和文档维护规则不要重复。

写入或更新完成后，读取 [completion-checklist.md](completion-checklist.md) 进行检查。

# 3. 写作原则

- 按任务目标或能力域提供加载路由，不要机械罗列所有文件
- 清楚区分需求文档、过程文档、长期 AI 检索文档和上下文入口
- 说明后续 AI 应先读什么、何时补充读什么、哪些文档暂时无关
- 不复述详细实现，入口只保留路由、跳过依据和误读边界；细节回到 AI 检索文档或源码确认
- 路由要能解释跳过依据，例如某类任务只需读入口和一个 AI 检索文档，不需要读取同目录全部需求文档
- 更新时优先删除失效路由、合并重复提示和替换过期边界，不因新增文档而扩写实现背景
- 新增或改名文档后，同步维护相对链接和加载路由

# 4. 适配项目工作流

维护上层入口时，只沉淀所在项目已有且会影响本层加载决策的文档治理差异。

不要为了补齐结构写入不存在的项目治理规则、通用回答要求或实现背景。不要把某个项目或某套 Skill 体系中的文件名写成通用前置条件，也不要复述上层承载文档已经说明的职责。

若项目没有对应工作流、临时工作区或 ADR 体系，只保留适用部分并说明本项目现状。
