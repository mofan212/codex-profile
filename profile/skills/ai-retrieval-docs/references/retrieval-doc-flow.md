# 1. AI 检索文档驱动流程

当触发时指定了 AI 检索文档，按本文件处理。

# 2. 向上查找入口

以此文档所在目录向上搜索最近的 AI 上下文入口：

- 找到入口后，读取 [context-entry-flow.md](context-entry-flow.md)，按 AI 上下文入口驱动流程处理
- 未找到入口时，先识别当前项目和文档树，再触发 🔴 CHECKPOINT：询问用户是否以该 AI 检索文档所在目录为起点创建或维护上层 AI 上下文入口

继续条件：用户确认上层入口目标路径和写入范围。

🛑 STOP：用户只要求分析当前文档，或未确认上层入口写入范围时，不创建入口。

如果用户只指定当前 AI 检索文档但没有要求扩大范围，不要默认读取入口范围内所有 AI 检索文档。

# 3. 直接更新当前检索文档

如果用户明确要求只更新当前 AI 检索文档，不需要扩大到入口路由范围：

1. 读取当前 AI 检索文档
2. 查找同目录同主题需求文档、设计文档或示例数据
3. 检查当前项目 git 变更和相关检索入口、隐藏规则、兼容边界
4. 读取 [writing-ai-retrieval-doc.md](writing-ai-retrieval-doc.md)，然后更新当前文档

不要把入口范围内所有文档作为默认读取对象。

更新后检查同目录入口或上层入口是否需要补充路由；只有路由缺失、文件改名或新增文档时才同步更新入口。
