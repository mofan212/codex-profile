# 1. 文档定位

本文用于 AI 快速检索「需求名称」的实现逻辑，描述当前 Workspace 中已经实现的代码结构、执行链路、关键行为、兼容边界和验证方式。

如需补充背景，可同时参考：

- 需求文档：说明原始目标和约束；生成文档时，如果同目录存在真实文件，使用 Markdown 相对链接，形如 `[文件名.md](文件名.md)`；如果只是候选文件名，用 `实际需求文档名.md` 表示，不要生成占位链接
- 相关设计文档：说明历史方案或上游背景；生成文档时，如果同目录存在真实文件，使用 Markdown 相对链接，形如 `[文件名.md](文件名.md)`；如果只是候选文件名，用 `实际设计文档名.md` 表示，不要生成占位链接

如果背景文件不存在或无法确认，不要生成占位链接。

# 2. 核心结论

用 2-5 段说明最终实现事实：

- 入口在哪里
- 核心行为是什么
- 关键数据结构或上下文是什么
- 与旧逻辑、生成逻辑或其他模块的关键边界是什么

# 3. 主要代码位置

核心实现：

- `module/src/main/java/.../ClassA.java`
- `module/src/main/java/.../ClassB.java`

测试覆盖：

- `module/src/test/java/.../ClassATest.java`

# 4. 执行入口链路

```text
EntryClass.entry(...)
  -> Service.method(...)
  -> Handler.handle(...)
  -> Utility.convert(...)
```

说明每一步的职责，特别是哪些步骤只做预处理，哪些步骤真正改变运行时行为。

# 5. 关键实现规则

- 规则 1：说明触发条件、输入、输出或副作用
- 规则 2：说明默认值、开关、兜底逻辑或跳过逻辑
- 规则 3：说明异常、错误码、消息或错误累积方式

# 6. 兼容性和边界

修改时不要破坏以下行为：

1. 历史默认入口行为
2. 出参、旁路调用或复用场景
3. 未匹配字段、未知配置或已有透传逻辑
4. 明确不属于本需求处理范围的场景

- 测试点 2

# 8. AI 快速检索提示

- 搜入口：`EntryClass`
- 搜核心处理器：`HandlerClass`
- 搜上下文：`ContextClass`
- 搜关键开关：`featureFlag`
- 搜错误码：`ERROR_CODE`

如果需要判断某个行为是否应与旧实现一致，优先查相关历史文档或测试，再回到当前核心类确认实际实现。
