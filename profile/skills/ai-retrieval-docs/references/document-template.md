# 1. 文档定位

本文用于 AI 快速检索「能力/需求名称」的关键入口、隐藏规则、兼容边界、验证入口和排查关键词。

如需补充背景，可同时参考：

- 需求文档：说明原始目标和约束；生成文档时，如果同目录存在真实文件，使用 Markdown 相对链接，形如 `[文件名.md](文件名.md)`；如果只是候选文件名，用 `实际需求文档名.md` 表示，不要生成占位链接
- 相关设计文档：说明历史方案或上游背景；生成文档时，如果同目录存在真实文件，使用 Markdown 相对链接，形如 `[文件名.md](文件名.md)`；如果只是候选文件名，用 `实际设计文档名.md` 表示，不要生成占位链接

如果背景文件不存在或无法确认，不要生成占位链接。

# 2. 快速索引

必须使用短列表，不使用表格；只保留能直接检索或定位源码的关键词、类名、方法名、路径、配置项、错误码或测试名。

排查信息优先压缩为索引项；只有跨模块、固定判断顺序且无法被索引、关键结论或验证入口表达时，才在具体文档中临时保留短小排查说明。

下列项目是候选项，不是必填项；没有事实来源或不能降低检索成本时删除。

- 入口：`EntryClass#entry`，路径 `module/src/main/java/.../EntryClass.java`
- 核心处理：`HandlerClass#handle`，路径 `module/src/main/java/.../HandlerClass.java`
- 关键对象：`ContextClass`
- 配置或常量：`featureFlag`、`ERROR_CODE`
- 排查关键词：`timeout`、`ERROR_CODE`，先看 `HandlerClass#handle` 和配置 `featureFlag`
- 代表性测试：`SomeTest#caseName`
- 跨项目位置：`other-module/src/main/java/.../ExternalClass.java`

# 3. 关键结论

本节可删除。只写命中写入门槛、不能通过索引快速确认、且后续容易误判的长期结论；每条只写一个事实，最多 0-6 条。

不要使用「潜规则」作为小标题。可选类型只是分类，不要求逐类填写；多个类型命中同一事实时合并成一条。兼容事实一句话能说清时写在本节，需要触发条件、入口或测试依据时写入 `# 5. 兼容性和边界`。

可选类型：

- 隐藏规则：源码分散、命名不直观或需要多步推理的规则
- 范围边界：全局模型、枚举、配置或通用逻辑支持更多场景，但当前能力只覆盖其中一部分
- 实现差异：代码命名、旧文档、需求说法或直觉判断与当前实际行为不一致
- 误判边界：不要把哪些旧逻辑、相邻模块、同名概念或通用能力混读
- 兼容事实：旧路径、新路径、开关、默认值或兜底行为的触发条件
- 诊断判断：后续排查时容易反复确认的判断依据

# 4. 关键链路

只在链路跨模块、跨项目、有隐藏分支或容易误判时保留，本节可删除。普通顺序调用不要展开。

```text
EntryClass.entry(...)
  -> HandlerClass.handle(...)              # 关键分支或跨模块跳转
  -> DownstreamService.execute(...)        # 隐藏边界或外部依赖
```

链路说明只写跳转原因、分支条件和误判点，不逐步解释每个方法职责。

# 5. 边界与兼容性

本节可删除。只写有代码、测试、配置、历史文档或用户明确结论支撑，且后续修改容易破坏的兼容边界。写入时说明触发条件、关联入口或测试依据；不能说明依据时删除。

可选类型：

- 历史默认入口、旧字段、旧配置或旧调用方兼容
- 出参、旁路调用、复用场景或透传行为
- 未匹配字段、未知配置、跳过分支、兜底逻辑或错误累积
- 明确不属于本能力处理范围的场景

# 6. 验证入口

常用验证命令：

```powershell
mvn -pl module -DskipTests compile
mvn -pl module test "-Dtest=SomeTest"
```

代表性覆盖：

- `SomeTest#caseName`：验证关键边界或隐藏规则
- `OtherTest#caseName`：验证兼容路径

不要罗列全部测试方法；只保留能帮助后续定位行为的测试。

如果需要判断某个行为是否应与旧实现一致，优先查相关历史文档或测试，再回到当前核心类确认实际实现。
