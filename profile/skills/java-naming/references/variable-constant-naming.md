# 1. 字段和变量命名规则

字段、参数和局部变量使用小驼峰。名称应简短但能表达业务含义，例如 `currentUserId`、`computedValues`、`orderItems`。不要以下划线或美元符号开头，不要使用只有自己知道的缩写。

局部短生命周期变量可以短一些；跨方法、跨类或公开 API 中出现的字段和参数必须更具体。循环下标可以使用 `i`、`j`，但业务数据不要使用单字符变量。

# 2. 布尔字段

布尔字段优先使用形容词或状态名，例如：

```text
enabled
active
deleted
visible
available
```

如果仓库或数据库字段已经稳定使用 `is_xxx` / `isXxx`，为了兼容可以保留，但不要主动为普通 Java 布尔字段新增 `is` 前缀体系。

# 3. 常量命名规则

类级常量和全局常量使用全大写下划线命名，例如：

```text
USER_MESSAGE_CACHE_EXPIRE_SECONDS
DEFAULT_PAGE_SIZE
MAX_RETRY_COUNT
```

常量名应表达完整业务含义，不要为了缩短而省略关键名词。方法内的 `final` 局部变量如果只是普通局部语义，可以继续使用小驼峰。

# 4. 变量和常量检查

最终确定字段、变量或常量名前，检查以下事项：

- 变量名是否足够具体；
- 是否避免了只有局部上下文才懂的缩写；
- 布尔字段是否表达状态，而不是机械添加 `is` 前缀；
- 常量名是否使用全大写下划线；
- 常量名是否表达完整业务含义。
